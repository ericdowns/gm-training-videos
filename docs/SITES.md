## Conventions

- **Slug** matches the Local by Flywheel site name (`~/Local Sites/<slug>/`).
- **Path type** = `Local` for standard `~/Local Sites/<slug>/app/public/wp-content/plugins/training-videos`. Anything else is flagged.
- **Installed version** = read from `training-videos.php` plugin header on the deployed copy, NOT the master repo.
- When a site is rebranded with diverged templates (like Xomox), flag it. The brand-theming card will reconcile these into a per-site config.

## How to refresh this file

The current scan was generated by walking `~/Local Sites/*/app/public/wp-content/plugins/training-videos/training-videos.php` and grepping the `Version:` header. Re-run that pattern when sites are added or updated.

## Why this exists (and what replaces it)

This is the gap that surfaces every time something changes in the master repo: there's no quick way to answer "which sites are on which version" without grepping the filesystem. The next-step fix is the **central registration system + dashboard** (cards on this repo) — once a site phones home on activation, the dashboard is the source of truth and this file becomes redundant.

Until then: keep this updated whenever you `cp -r` the plugin to a site.
